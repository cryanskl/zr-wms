# WMS Price Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build slice 9-C: product price read/write with role restrictions.

**Architecture:** Keep price endpoints inside the product module because `price` is a 1:1 product child table. Backend raw SQL reads/upserts `price`; frontend shows a price section in product detail.

**Tech Stack:** TypeScript, NestJS, PostgreSQL raw SQL via `pg`, React, Vite, Ant Design, TanStack Query.

## Global Constraints

- Do not modify authoritative SQL files.
- Price GET requires ADMIN or BOSS.
- Price PUT requires BOSS only.
- Inventory and stock movement tables are not touched by this slice.
