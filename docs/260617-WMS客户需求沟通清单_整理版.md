# WMS 客户需求沟通清单（整理版）

> 用途：当面/线上与客户沟通需求，理清现状与期望。
> 结构：按沟通顺序分四块 —— **① 痛点与目标 → ② 现状了解 → ③ 打算与期望 → ④ 设备与环境**。
> 用法：每块按从上到下问即可；时间紧就只问下方「30 分钟精简版」。

---

## 开场（先对齐定位，再开始问）

可以这样开场，把双方预期对齐：

> 「我们第一版不打算一上来做个很复杂的 WMS，而是先把你们现在微信里那些库存信息，变成**能查、能追溯、能确认**的结构化数据。目标不是功能多，而是让库存更准、找货更快、出入库有记录、责任能追溯。所以今天主要想搞清楚你们现在是怎么干的、最头疼什么、想达到什么效果。」

---

## ⏱ 30 分钟精简版（时间紧就只问这些）

1. 现在为什么想上系统？最头疼的是什么？如果第一版只解决一个问题，先解决哪个？
2. 现在库存数据在哪——微信、Excel，还是已有系统？有没有现成的产品清单/库位表/库存表？
3. 产品编码是否唯一、有没有不规范写法？「带管子」「铝板/防尘罩」这类怎么算？
4. `50*4件` 是 200 件还是「4 包×每包 50」？单位有几种？
5. 一个产品会不会放多个库位？一个库位会不会放多个产品？
6. 出入库有哪些实际场景？要不要移库、盘点、库存调整？
7. 出入库是否必须**人工确认后**才生效？谁能改库存、谁最终确认？
8. 现在用微信、企业微信还是飞书？愿不愿意切到飞书/企业微信？
9. 现在用的是哪家 ERP？有没有 API、能不能读写？库存以谁为准？
10. 想本地部署还是云?仓库网络稳不稳?现场有没有扫码枪/条码?
11. 初始库存数据从哪来、准不准、能不能导 Excel?
12. 第一版必须上线哪些、哪些二期再做?能不能先选一个货架/区域试点?

# ① 痛点与目标（先问「为什么」）

这部分决定整个系统的重心，避免做出来「换了记录形式，没解决真问题」。

1. 为什么现在想上系统？是出了什么事，还是单纯想规范化？

销售问成品，半成品？缺什么。

查询。

提醒，预警。（提醒）匹配的是订单。

1. 现在**最头疼**的是哪几个（可多选）：
   -  不好搜 / 容易漏看

   3 个人录入，每个人叫法不一样。排查记录不准确，时间不精确。3 个仓库。

   不会同时。

   - 经常算错库存、数量不准

   原材料价格。管理配件数量。

   - 多人记录导致数据不一致
   - 谁出库、谁改了库存说不清，没法追责
   - 找货慢
   - 产品编码 / 库位记录不统一

   可能有bias，编码多可能有别名

2. 如果第一版**只能解决一个问题**，最想先解决哪个？

3. 上系统后希望达到什么效果（库存更准 / 找货更快 / 有记录可追责 / 能导报表 / 减少微信反复确认）？

4. 怎么算这套系统做成功了？（用来后面定试点验收标准）

# ③ 打算与期望（聊「想怎么做」）

## 3.1 录入方式的意愿（微信 / 飞书 / 企业微信）

1. 现在大家主要用微信、企业微信还是飞书？
2. 愿不愿意切到飞书 / 企业微信？还是希望继续用微信？
   > 提示：个人微信没有正规机器人接口，做生产入口有封号风险；飞书 / 企业微信才稳妥，飞书还能用「带按钮的确认卡片」。
3. 能否接受「**发消息 → 系统识别 → 人工确认 → 生效**」，而不是发一句直接改库存？
   > 强烈建议加人工确认这步，避免误识别、误扣库存、责任不清。
4. 是否希望手机操作 + 一个电脑端管理后台？
5. 要不要拍照上传 / 语音转文字？
6. 每次库存操作后要不要在群里自动通知？管理员要不要在手机上审批确认？
7. 要不要保留原始微信文本，方便追溯？

## 3.2 ERP 与 WMS 的边界（关键决策）

> 原则参考：**ERP 管业务单据和财务账，WMS 管仓库现场、库位和实物库存**。但「库存以谁为准」必须客户拍板。

1. 产品主数据以 ERP 为准，还是 WMS 可以自己维护？
2. 库存数量以谁为准？两边对不上时听谁的？
3. WMS 要不要把出入库结果**回传** ERP？ERP 要不要给 WMS 下发采购单 / 销售单 / 领料单？
4. 第一版**先不对接 ERP、只做独立库存台账**，后续二期再接，行不行？

## 3.3 权限与责任

> 建议：流水不真删，用「作废 / 冲正」保留记录。

1. 谁能查库存 / 改库存数量 / 做出库 / 做入库 / 做库存调整 / 作废记录？
2. 要不要管理员审核？普通员工能不能看到全部库存？
3. 每次操作是否记录操作人和时间？操作错了能不能撤销、谁有权冲正？

## 3.4 查询、统计与导出

> 第一版建议至少支持：按产品查、按库位查、按时间查流水、按操作人查流水、导出 Excel。

1. 最常用的是按产品查，还是按货架查？
2. 要不要模糊搜索（输 `399 151` 能找到 `4B0 399 151`）？
3. 要不要查：某产品在哪些货架 / 某货架上有哪些产品 / 某产品历史流水 / 按时间 / 按操作人？
4. 要不要：导出 Excel、低库存自动提醒、日 / 周 / 月报、呆滞库存统计、库位利用率？

## 3.5 追溯与质量要求

> 看着像汽配零件号——汽车行业常有批次追溯 / 召回要求，这块要提前定，后补很痛。

1. 要不要记录采购日期 / 生产日期 / 入库日期？



1. 如果暂时不做，需在需求里写明：第一版仅记录产品、库位、数量和操作流水，不做批次追溯。



## 3.6 第一版范围（MVP）

1. 第一版**必须上线**哪些功能？哪些可以二期再做？
2. 能否接受第一版先用网页系统、**不做 3D 展示**？
3. 能否接受第一版先做「文本识别 + 人工确认」、不直接自动改库存？
4. 能否先选**一个仓库或一个区域试点**？试点成功的标准是什么（库存更准 / 找货更快 / 记录更全 / 减少微信反复确认 / 能导报表）？

# ④ 设备与环境（落到「怎么部署、用什么」）

## 4.1 部署与网络

1. 希望本地部署（办公电脑 / 小服务器 / Mac mini）？
   > 制造/汽配客户常要求数据不出厂，倾向本地部署。
2. 仓库现场网络稳不稳？有没有固定公网 IP？
3. 要不要支持外地人员、或仓库外用手机访问？
4. 要不要每日自动备份 / 断电保护 / UPS / NAS？

## 4.2 设备

1. 要不要扫码枪 / 手机扫码 / PDA 手持终端？
2. **产品上现在有没有现成的条码可以扫？**（没有的话扫码方案要换思路）
3. 要不要打印产品标签 / 库位标签 / 出入库单？
4. 要不要仓库大屏展示？

## 4.3 初始数据与上线准备

> 建议上线前完成：产品清单整理 → 库位清单整理 → 初始盘点 → 异常数据清洗 → 导入测试 → 客户确认初始库存。

1. 第一批库存数据在哪？有没有现成 Excel？里面含不含产品编码 / 数量 / 库位？
2. 现有数据准不准？编码规不规范？有没有同一产品多种写法？
3. 上线前要不要做一次全仓盘点？初始库存由谁确认、要不要签字？
4. 没有现成 Excel 的话，能否先按库位逐步录入、或先选一个货架/区域试点？
5. 要不要导入历史出入库记录、产品图片 / 规格？要不要保留原 Excel 做备份？
