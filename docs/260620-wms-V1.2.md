这个是和客户沟通后的问题和答案。

客户是做汽配的。

总得来说，客户的核心需求是，能又快又准的查到产品对应的库存。

所以我想给客户提供的就是，记得快，记得准，查得快，查得准。



客户遇到困境：

1.有五个人录入，每个人叫法可能不一样。所以一些产品会有多个，别名。要设置 alias，或者标签。也要和随时加进来的订单绑定。

2.先前是微信群里记的，有时候查记录时间和产品对不上，不好追溯



客户的额外需求：

1.产品要能看到，上传图片（不确定是否要多图片）。

2.日志要有产品变动，时间，操作人员。有冲突能报警。冲突可能是同时修改。或者是库存不足报警。

3.客户有三个仓库，之后可能会加仓库。每个仓库有8排货架， 可能会加货架。每排货架有24列，每列货架三层，每层可能前后各放一筐或者就整体放一筐，然后前或后并排放两筐或三筐（比如说正常是每列）。这些仓库数，排数，列数，层数，能放的框数都要能调整。几号仓库-几号货架-第几列-第几层-前后（设为 ab）。

4.需要考虑设计的是，货架要有模板。然后仓库也可以保存模板复制。

5.客户的产品分为，原材料，半成品和成品。加起来我们先设计两万个种类。每个数量设计能存一万件。

6.但这些原材料，半成品和成品可能在不同仓库，不同货架的不同位置。需要好追溯。

7.客户可以人为增删改查数据。

8.可以做伪 3D

9.可以有智能体，一个是计算订单目前匹配的原材料，成品，半成品的推衍。一个是总结每日，每周，每月的进度。汇报和提醒事项。导出 Excel、低库存自动提醒、日 / 周 / 月报、呆滞库存统计、库位利用率

10.原材料，半成品和成品之间的配方。

11.原材料，半成品和成品的双链。比如我选择原材料，会显示多少库存，能做多少半成品，能做多少成品。当然这个需要点击去算，而不是实时渲染显示。原材料和半成品是多对多，半成品和成品是多对一。

12.客户想自定义。比如7L0 199 131F。那么7L0 199 131F-1 是半成品，7L0 199 131F-1-1 是原材料。这个和双链类似。但是我觉得没之前双链好，但这也是一种双链。你可以帮我想一下半成品和原材料的命名方式。

13.入库有损耗，退货，原材料，配件，半成品，成品。出库有成品，半成品，配件，损耗。

14.移库很少。但是客户会自己控制。可能合并到别的库位。客户会手动修改。

15.配件，原材料，半成品，成品都要有照片。

16.关于查询。可以模糊搜索。按产品查。按标签和备注查。要能查某产品在哪些货架 / 某货架上有哪些产品 / 某产品历史流水 / 按时间 / 按操作人？

17.记录采购日期 / 生产日期 / 入库日期？



不明确的：

1.原材料价格，订单价格，出库价格想怎么安排

2.有些地方有消防栓，怎么设计。

3.三列放 2 框，怎么设计标记。

4.原材料合成半成品，合成成品。是否需要把公式，配方和流程写在备注里。是否有保密性。

5.权限设置。哪些功能不能开放给谁。

6.订单长什么样？

7.一筐能不能装多个产品?
