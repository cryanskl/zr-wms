# WMS 界面设计简报（给 Claude Design）

## 要做什么

为一套**汽配仓库管理系统**设计界面。这是**内部后台系统**（不是面向消费者的产品），仓库工作人员每天高频使用。
流程：先做关键页面的高保真设计（mockup）→ 定稿后连成可点击原型（prototype）。

## 产品定位

- 多仓库存管理。三个仓库、大量 SKU（原材料 / 半成品 / 成品 / 配件）。
- 客户**头号诉求：查得快、查得准**。搜索是系统的心脏。
- 从微信群记账升级而来，用户不是技术人员 → 界面要**直白、少术语、操作步骤短**。

## 用户与角色（三种，菜单和可见功能不同）

- **操作员**：每天干活的人。高频做 搜索、出入库、移库、查库存。界面要**快、移动端好用**。
- **管理员**：管产品主数据、BOM、仓库结构。
- **老板 / 财务**：看报表、看/改价格。多为只读 + 价格。

设计时**主要服务"操作员"的高频场景**。

## 设计约束

- **响应式网页**：手机浏览器 + 桌面都要用（手机加桌面图标即用，不做独立 App / 小程序）。**核心操作（搜索、出入库）必须在手机上顺手。**
- **中文界面**。
- 视觉风格**贴近 Ant Design**（前端用 AntD 实现，设计接近它能让落地省事）。简洁、信息密度适中的后台风格，不要花哨。
- 仓库场景：数据多、表格多，但要**可扫读**，别堆砌。

## 要设计的页面（按优先级，先做前三个）

**P0（先做）**

1. **搜索 / 查询页（首页）**：醒目的搜索框，支持按 名称 / 别名 / 编号 / 路径别名 模糊搜；结果列表显示产品 + 它在哪些库位有多少。还要能反查"某库位上有哪些产品"。带"最近用过的产品"快捷入口。
2. **产品详情页**：产品信息、类型、属性、别名、图（≤3 张）、它在各仓库各库位的库存（总 / 可用 / 冻结）、它的 BOM、路径别名。
3. **出入库操作页**：入库 / 出库 / 移库表单——选产品（带搜索）、选库位、填数量、填原因。库存不足要醒目拦截提示。**移动端优先**。

**P1**

4. **库存看板**：跨仓库存总览 + 低库存预警列表。
5. **订单页**：采购 / 生产订单列表 + 详情；生产订单要能展示"缺料推衍"结果（差几个原料，逐层列出）。
6. **盘点页**：录实盘、看差异、生成调整。

**后续**：仓库 / 货架可视化（伪 3D 货架图）、报表、价格管理。

## 交互要点

- 任何"选产品"的地方都要**带搜索 / 模糊匹配**（这是最高频的动作）。
- 出库时若库存不足，要**醒目拦截提示**（可由管理员确认强制出库）。
- 操作类页面尽量**一屏完成、少跳转**。
- 三角色看到的导航菜单不同：操作员只看到日常操作，老板看到报表 / 价格。

## 不归这里设计的

数据库、API、后端逻辑都已另行完成。本简报只关心**界面长什么样、怎么操作**。
